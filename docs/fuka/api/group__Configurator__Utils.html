Detailed Description

Utility functions to enable parsing and saving Configurator files currently based on the BOOST proprietary INFO format. In principle this can be changed easily by changing the include to another BOOST parser, however, this is untested.

append_map

Here we check to see if there is an enabled value in the boolean storage array that corresponds
to a key/value pair that doesn't already exist in the parial_map. IFF one is found, we add it
to a new map and return it.
This is primarily useful for developers so they can have non-standard stages enabled in the
config file which aren't deleted after a run.

[input]full_map: Full map of possible elements
[input]partial_map: subset of elements from full_map
[input]storage: array of toggles based on full_map
Returns
map: possibly an appended version of partial_map.
